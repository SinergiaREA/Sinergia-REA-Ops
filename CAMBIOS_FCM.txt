🚀 CÓMO USAR
─────────────────────────────────────────────────────────────────

1️⃣  VERIFICACIÓN RÁPIDA (30 segundos)
   • Inicia sesión en la app
   • Abre consola: F12
   • Escribe: window.__FCM_Tests.quickCheck()
   • Presiona ENTER

2️⃣  VER PANEL VISUAL (Admin)
   • Aparecerá botón "🔍 FCM" en la barra superior
   • Haz clic para abrir modal con detalles
   • Verás estado, token, UID y cronología

3️⃣  ANÁLISIS PROFUNDO
   • window.__FCM_Tests.runAll()
   • Ver: tests/README.md

📊 INTERPRETACIÓN DE RESULTADOS
─────────────────────────────────────────────────────────────────

✅ TODO VERDE = FCM FUNCIONAL
   • Usuario autenticado
   • FCM inicializado: ACTIVE
   • Token generado
   • Permisos: granted
   • UID disponible
   → Notificaciones push FUNCIONARÁN

⚠️  WARNING = PROBLEMAS DETECTADOS
   • PERMISSION_DENIED → Usuario rechazó permisos
   • TOKEN_ERROR → Error en Firebase
   • TOKEN_NULL → VAPID Key incorrecta

❌ TODO ROJO = FCM NO FUNCIONA
   • Verificar consola con [FCM] filter
   • Ver tests/README.md sección "Problemas Comunes"

───────────────────────────────────────────────────────────────────

🔍 DEBUGGING
─────────────────────────────────────────────────────────────────

En la CONSOLA (F12):
   • Busca [FCM] para ver todos los eventos
   • Cada línea muestra exactamente qué sucedió
   • Timestamps precisos para diagnosticar

window.__FCM_DEBUG contiene:
   • status: "ACTIVE" / "ERROR" / etc
   • token: Token FCM completo
   • uid: UID del usuario
   • logs: Array de eventos
